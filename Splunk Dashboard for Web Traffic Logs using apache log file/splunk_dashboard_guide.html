Step 2 — Upload Your Dataset
Getting apache_logs.json into Splunk
⚠️
Critical: When Splunk asks for a Host value during upload, do NOT type webserver. Enter your actual machine/computer name instead. Every query in this project uses host="webserver" as a placeholder — replace it with your hostname everywhere.
1
Open Splunk Web and go to Settings
In the top navigation bar, click Settings → then click Add Data.
2
Choose Upload method
On the "Add Data" page, click the Upload tile (the first option). This lets you upload a file from your computer.
3
Select your file
Click Select File and choose apache_logs.json from your computer. Click Next.
4
Set Source Type to _json
In the "Set Source Type" screen, search for and select _json. This tells Splunk the file is in JSON format. Click Next.
5
Set the Host field to YOUR machine name
In the "Input Settings" screen, find the Host field. Change it from the default value to your computer's hostname. Set Index to main. Click Review.
6
Submit and verify
Click Submit. Splunk will confirm the file is indexed. You can then click Start Searching to verify data is visible.

Task 0 — Time Range Filter
Shared time picker used by all panels

Before adding any panels, set up a shared Time Range input. Every panel will reference this token so they all filter together when you change the time.

1
Enter Edit mode
Open your new dashboard and click the Edit button (top right corner).
2
Add a Time input
Click + Add Input at the top of the dashboard → Select Time.
3
Configure the time input
Click the pencil (✏️) icon on the Time input. Set Label to Time Range and Token to time_range. Click Apply.
4
Add a Submit button
Click + Add Input again → Select Submit. This adds a button to apply the time filter.
✅
For every panel you create below, always set its Time field to use the shared token $time_range$. This ensures all panels filter together.
📊
Task 1 — KPI Summary Panels
Single Value panels showing key numbers at a glance

For each panel below: click + Add Panel → New → Single Value → set Time to $time_range$ → set the title → paste the query → click Apply.

📌 Single Value
Total Web Activities
Counts all events — every HTTP request in the log.
SPL Query 
source="apache_logs.json" host="webserver" sourcetype="_json"
| stats count AS "Total Web Requests"
⚠️
Replace webserver with your machine name.
📌 Single Value
Successful Responses
Counts HTTP 200 GET requests — successful page loads.
SPL Query 
source="apache_logs.json" host="webserver"
sourcetype="_json" method=GET status=200
| stats count AS "Successful Responses"
📌 Single Value
Client Errors (4xx)
Bad requests from the client side — 404 Not Found, 403 Forbidden, etc.
SPL Query 
source="apache_logs.json" host="webserver"
sourcetype="_json"
| where status>=400 AND status<500
| stats count AS "Client Errors"
📌 Single Value
Server Errors (5xx)
Server-side failures — 500 Internal Error, 503 Unavailable, etc.
SPL Query 
source="apache_logs.json" host="webserver"
sourcetype="_json"
| where status>=500 AND status<600
| stats count AS "Server Errors"
🐛
Bug fix by Somesh: The original lab used status>=400 for 5xx — that's wrong. Use status>=500 to correctly count server errors only.
📈
Task 2 — Bar Chart Panels
Visualizing top URIs and top IP addresses

For each panel below: click + Add Panel → New → Bar Chart → set Time to $time_range$ → set the title → paste the query → click Apply.

📊 Bar Chart
Top Requested URIs
Which pages/endpoints are hit the most? Shows top 20 URLs by request count.
SPL Query 
source="apache_logs.json" host="webserver"
sourcetype="_json"
| stats count AS "Hits" by uri
| sort -Hits
| head 20
💡
Added by Somesh: | sort -Hits | head 20 limits to top 20 URIs so the chart stays readable.
📊 Bar Chart
Top Users by IP Address
Which IP addresses generate the most traffic? Shows top 20 IPs.
SPL Query 
source="apache_logs.json" host="webserver"
sourcetype="_json"
| stats count AS "IP" by ip
| sort -IP
| head 20
💡
Added by Somesh: | sort -IP | head 20 shows only the top 20 most active IPs.
🌍
Task 3 — Geographic Map Panel
Choropleth world map of traffic by country

This panel converts IP addresses into countries and colors a world map based on traffic volume per country. Click + Add Panel → New → Choropleth Map → set Time to $time_range$ → set title → paste query → click Apply.

🗺️ Choropleth Map
Web Traffic by Client IP Addresses
Uses Splunk's iplocation command to convert IPs into countries, then maps them geographically using geom geo_countries.
SPL Query 
source="apache_logs.json" host="webserver"
sourcetype="_json" method=GET
| table ip
| iplocation ip
| stats count by Country
| geom geo_countries featureIdField="Country"
CommandWhat it does
table ipKeeps only the IP column, discards other fields
iplocation ipAdds Country, City, Lat, Lon columns by looking up each IP
stats count by CountryCounts requests per country
geom geo_countriesMaps country names to polygon shapes for the choropleth map
⚠️
The geom command requires Splunk Enterprise (not Free edition). It uses the built-in geo_countries lookup that ships with Splunk Enterprise 6.5+.

Troubleshooting
Common issues and how to fix them
ProblemLikely CauseFix
No results in any panel host= doesn't match your machine name Run hostname in your terminal and replace webserver with that value in every query
5xx panel shows same count as 4xx Using wrong status filter Make sure the 5xx query uses status>=500 AND status<600, not status>=400
Map panel shows no data iplocation lookup may be disabled Go to Settings → Lookups and check that MaxMind GeoIP lookup is enabled
geom command not recognized Using Splunk Free edition Upgrade to Splunk Enterprise — geom is not available in the free version
Bar charts show 0 or are empty Dataset not indexed correctly Go to Search and run: source="apache_logs.json" — if no results, re-upload the file and check sourcetype is _json
Time filter not working Token name mismatch Verify the Time input token is exactly time_range (no spaces, lowercase)
